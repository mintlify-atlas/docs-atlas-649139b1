## Tools Used Across Writeups

Every machine in this section was attacked from a **Kali Portable** environment — a fully self-contained Docker-based Kali Linux station with GPU acceleration and 100+ pre-installed pentesting tools. You can read more about it at [/kali-portable/overview](/kali-portable/overview).

The core toolset across these writeups includes:

- **Nmap** — Service and version scanning (`-sC -sV --min-rate`)
- **Hydra** — Dictionary-based brute force against SSH
- **Gobuster** — Web directory and file enumeration
- **Metasploit** — Module-based exploitation (vsftpd backdoor)
- **stegseek / steghide / exiftool** — Steganography and metadata analysis
- **GTFOBins** — Reference for sudo/SUID privilege escalation

<Note>
All writeups use `172.17.0.2` as the target IP — the default address Docker assigns to the first container on the `docker0` bridge network. Your machine may receive the same IP, but always confirm with the output of `./auto_deploy.sh`.
</Note>

## Launching Your Environment

Before attacking any machine, start your Kali Portable session in a second terminal window:

```bash
./run-kali.sh normal
```

Then deploy the target machine in the first terminal by running the provided deploy script (see individual writeups for machine-specific commands). When you are done, press `Ctrl+C` in the deploy terminal to stop and remove the container.

<Warning>
Only test these techniques on machines you are **authorised** to attack. DockerLabs machines are intentionally vulnerable and designed exclusively for educational practice. Never run these attacks against systems you do not own or have explicit written permission to test.
</Warning>

</Tip>
